Iterative Loop Architecture

Music video creation is iterative — the system supports feedback loops, backward transitions, and revision at every stage

Core Principle: This is NOT a linear pipeline. Every stage can loop back to any previous stage. Quality gates verify output at every transition. The system continuously refines until the creative vision is achieved.

Revision Trigger Examples

Quality Gate rejects visual consistency
  → Backward to Visual Gen → regenerate inconsistent scenes only
 
User dislikes storyline after seeing storyboard
  → Backward to Creative Direction → revise storyline
  → Cascade storyboard + visuals marked stale, re-process affected scenes
 
Quality Gate detects lipsync artifacts in post-production
  → Backward to Visual Gen → regenerate that specific clip
  → Forward re-assemble with new clip only
 
Downstream Video Gen finds storyboard scene too vague
  → Escalate to Storyboard → add detail to that scene
  → Forward regenerate only that scene's visuals

Quality Gate Agent NEW

Dedicated verification at every stage transition — can PASS, SOFT FAIL (suggest changes), or HARD FAIL (auto-reject)

How it works: After each stage completes, the quality-gate-agent evaluates the output before the orchestrator advances. Three outcomes: PASS (proceed), SOFT FAIL (notify user with suggestions), HARD FAIL (auto-reject and re-process with feedback).
Stage N completes → output saved (versioned)
    │
    ▼
Quality Gate Agent evaluates output:
    │
    ├─ ✓ PASS → proceed to Stage N+1
    │
    ├─ ⚠ SOFT FAIL → suggest improvements
    │    ├─ User approves anyway → proceed
    │    └─ User agrees to revise → re-process Stage N
    │
    └─ ✗ HARD FAIL → auto-reject
         └─ Re-process Stage N with failure feedback
             (max 3 retries before escalating to user)

Revision Manager NEW

Every stage output is versioned — compare, revert, branch. Cascade rules propagate changes downstream smartly.

Cascade Rule: When a stage is revised, all downstream outputs are marked stale. Only affected scenes are re-generated — not the entire downstream pipeline. This keeps iteration fast and cost-efficient.

Version Tree Example

project_123/ ├── music_analysis/ │ └── v1/ analysis.json ← initial analysis (stable) │ ├── storyline/ │ ├── v1/ storyline.json ← auto-generated │ └── v2/ storyline.json ← user revised mood from "dark" to "dreamy" │ ├── storyboard/ │ ├── v1/ storyboard.json ← from storyline v1 (STALE — storyline changed) │ ├── v2/ storyboard.json ← cascaded from storyline v2 │ └── v3/ storyboard.json ← user swapped scene 3 manually │ ├── visuals/ │ ├── v1/ scenes/ ← from storyboard v1 (STALE) │ ├── v2/ scenes/ ← from storyboard v3 (only scenes 3,7,12 regenerated) │ └── v3/ scenes/ ← quality gate rejected scene 5 → regenerated │ └── final/ ├── v1/ video.mp4 ← first assembly └── v2/ video.mp4 ← lipsync improved on scenes 2,4

Cascade Propagation Logic

Stage N revised (new version created)
    │
    ▼
Revision Manager analyzes impact:
    │
    ├─ Diff old vs new output
    ├─ Identify which downstream scenes are affected
    ├─ Mark affected downstream outputs as STALE
    └─ Queue re-generation for affected scenes ONLY
 
Example: Storyline v2 changes mood for chorus section
  → Only chorus scenes in storyboard re-built
  → Only chorus visuals re-generated
  → Verse scenes untouched (saves cost + time)

Risk Assessment

High
Visual Inconsistency Across Scenes

Each clip independently generated. Mitigation: Ref images + style lock + color matching + quality gate auto-rejects inconsistent scenes for targeted re-generation.

High
Infinite Revision Loops

Quality gate keeps rejecting, system loops forever. Mitigation: Max 3 auto-retries per stage, then escalate to user. Configurable thresholds. User can override any quality decision.

Med
Cascade Explosion

One upstream change triggers full downstream re-generation. Mitigation: Smart cascade — diff analysis identifies only affected scenes. Unchanged content is preserved.

Med
High Cost from Re-generations

Iterative loops multiply API costs. Mitigation: Cost tracking per revision, budget limits, quality scoring to minimize retries, cost-aware model routing.

Med
Long Generation Times

3-5 min video = many clips, loops add time. Mitigation: Parallel generation, targeted re-gen only, preview assembly, SSE progress feedback.

Med
Quality Gate Calibration

Too strict = endless loops, too lenient = poor output. Mitigation: Configurable thresholds, user override capability, feedback learning over time.
